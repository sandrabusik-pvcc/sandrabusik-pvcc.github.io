    All About Github

    Q. What is a version control system?
    A. A version control system is aclass of systems responsible for managing changes to computer programs, documents, large web sites, or other collections of information.
    Q. What is Git?
    A. Git is a distributed version control system that tracks changes in any set of computer files, usually used for coordinating work among programmers who are collaboratively developing source code during software development. Its goals include speed, data integrity, and support for distributed, non-linear workflows.
    Q. What is a Github?
    A. GitHub, Inc. is a developer platform that allows developers to create, store, manage and share their code. It uses Git software, providing the distributed version control of Git plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project.
    Q. What is the difference betweeen Git and Github?
    A. Git is a version control system that allows developers to track changes in their code. GitHub is a web-based hosting service for git repositories.
    Q. Who started Github and how was it started?
    A. Chris Wanstrath and PJ Hyett were webcoders who were allowed after tough persuading to change the sourcecode to programs and this proved frustrating, so the two worked overtime to develope a platform where you could change sourcecode easily called Github
    Q. What company owns it now?
    A. Microsoft
    Q. How much does a Github account cost?
    A. $49/month

      Github terms:

      repository: a repository is a data structure that stores metadata for a set of files or directory structure.
      commit:save
      fork:a new repository that shares code and visibility settings with the original “upstream” repository.
      push:how you transfer commits from your local repository to a remote repo.
      pull requests:let you tell others about changes you've pushed to a branch in a repository on GitHub.
      workflows:a configurable automated process that will run one or more jobs.
      issues:GitHub Issues are items you can create in a repository to plan, discuss and track work.
      raw button:you can view or copy the raw content of a file without any styling.
      blame button:git blame command is used to examine the contents of a file line by line and see when each line was last modified and who the author of the modifications was.
