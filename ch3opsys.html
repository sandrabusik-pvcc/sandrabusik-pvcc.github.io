Operating Systems Terminology

TERM

DEFINITION
Operating System Software that controls the overall operation of a computer.
job queue In batch processing systems, the jos residing in mass storage wait for execution in this.
FIFO First-in, first-out
interactive processing when a program being executed can carry on a dialogue with the user
time-sharing operating systems that provide service to multiple users at the same time.
multiprogramming Time is divided into intervals and then the execution of each jo is restricted to onl one interval at a time. At the end of each interval, the current job is temporarily set aside and another is allowed to execute during the next interval. By rapidly shuffling the jobs back and forth in this manner, the illusion of several jobs executing simultaneously is created.
application software Programs for performing tasks particular to the machine's utilization.
system software performs those tasks that are common to computer systems in general
utility software Within system software, software units known as utility software.
graphical user interface (GUI) Files and prorams are represented pictorially on the display as icons.
shell Older interfaces where communication with users was through textual messages.
kernel Internal part of an operating system. Contains those software components that perform the very basic functions required by the computer intallation.
device driver A component of the kernel, a software unit that communicated with the controllers to carry out operations on the peripheral devices attached to the machine.
file manager A softare component in the kernel that coordinates the use of the machine's mass storage facilities. It maintains records of all files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or etensions to existing files.
memory manager A software component in the kernel which coordinates the machine's use of main memory.
virtual memory When the total main memory space required exceeds the space actually available in the computer the memory managermay create the illusion of additional memory space by rotating prorams and data back and forth between main memory and mass storage
boot strapping How the operating system gets started. This procedure transfers the operating system from mass storage into main memory.
ROM A small portion of compuer's main memory where the CPU expects to find its initial poram is constructed from Read Only Memory because its contents can be read bur not altered.
deadlock condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to the other.
spooling Holding data for output at a later but more convenient time. This hold of data creates the illusion of having multiple resources.
super user (or administrator) Administrator. The user who has highly privileged access to the operating system who can alter settings in the operating system , modigy critical software packaes, adjust priveges granted to other users, and perform a variety of other maintnenace activities that are denied normal users.
multi-factor authentication One solution to the pervasive problem of humans usin weak passwords to secure their computer accounts is to require more than one piece of evidence to gain access, such as an ATM card and a special access code text messaed to the user's smartphone.
